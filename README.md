# 🔗 Backlink Checker

Инструмент для проверки гест-постов: находит ссылки на ваши домены и проверяет индексацию страниц в Google.

**Продакшн:** https://backlink-checker-production.up.railway.app

---

## Что умеет

- Проверяет наличие ссылок на один или несколько доменов одновременно
- Показывает **anchor text** и тип ссылки (dofollow / nofollow / sponsored / ugc)
- Проверяет **индексацию в Google** через DataForSEO API
- Результаты появляются в реальном времени по мере проверки
- Сохраняет **историю** всех проверок
- Экспорт результатов в **CSV**
- Кнопка **"Перепроверить без ссылки"** — повторная проверка только тех URL, где ссылка не найдена (удобно когда нужно докинуть домены)
- Кнопка **"Перепроверить ошибки"** — повторная проверка URL с ошибками индексации

---

## Статусы ссылки

| Бейдж | Значение |
|---|---|
| ✓ Найдена | Ссылка на домен найдена в HTML страницы |
| ✗ Нет | Ссылка не найдена |
| ⚠ JS-рендеринг | Страница загрузилась, но ссылка возможно скрыта за JavaScript — проверить вручную |
| ⊘ Недоступен | Сервер не смог достучаться до сайта (DNS-ошибка) |
| ⊘ Таймаут | Сайт не ответил за 20 секунд |
| ⚠ Ошибка | Другая техническая ошибка |

## Статусы индексации

| Бейдж | Значение |
|---|---|
| ✓ В индексе | Страница найдена в Google по запросу `site:url` |
| ✗ Не в индексе | Страница не найдена в Google |
| ⚠ Нет баланса | Закончился баланс DataForSEO — пополнить на app.dataforseo.com |

## Стек

- **Backend:** Python 3.11, FastAPI, httpx, BeautifulSoup4
- **Frontend:** Vanilla HTML/CSS/JavaScript (без фреймворков)
- **База данных:** SQLite (история проверок)
- **Индексация:** DataForSEO SERP API (`site:` запрос)
- **Деплой:** Railway (Docker)
- **Обновления:** автодеплой при пуше в `main` ветку GitHub

```bash
git clone https://github.com/101rtpteam/backlink-checker.git
cd backlink-checker
cp .env.example .env   # вставь DFS_CREDENTIALS
./start.sh
```

## Google Apps Script — автоматизация через Google Sheets

Скрипт `google-apps-script.js` подключается к таблице гест-постов и автоматически проверяет все ссылки через Backlink Checker API.

### Что делает скрипт

Читает строки в листе **Guestposting**, где:
- Колонка **K** = `Published` (статья опубликована)
- Колонка **L** содержит URL гест-поста

Для каждой такой строки вызывает API и заполняет:

| Колонка | Что записывает |
|---|---|
| **M — Exists** | `Yes` / `No` / `404` / `Other` |
| **N — Index** | `Yes` / `No` / `Нет баланса` |
| **O — Dofollow** | `Yes` / `No` |
| **P — Anchor** | текст анкора ссылки |

Ячейка M подсвечивается цветом: 🟢 `Yes`, 🔴 `No` / `404`, 🟡 `Other`.

### Установка

1. Открой таблицу → **Расширения → Apps Script**
2. Вставь содержимое файла `google-apps-script.js`
3. Сохрани (Ctrl+S) → перезагрузи таблицу
4. В меню появится **🔗 Backlink Checker**

> ⚠️ При первом запуске Google попросит выдать разрешения — нужно подтвердить.

### Использование

После установки в таблице появляется меню **🔗 Backlink Checker** с двумя кнопками:

**Проверить все Published** — проходит по всем строкам с `Published` в колонке K и запускает проверку. Подходит для первичной проверки или полного обновления данных.

**Проверить выделенные строки** — выдели нужные строки и запусти. Удобно когда нужно перепроверить конкретные URL после исправления ошибок.

### Важно: настройка колонки M

В колонке **M** должна быть настроена проверка данных (dropdown) с допустимыми значениями:
```
Yes, No, 404, Other
```
Без этой настройки скрипт всё равно работает, но рекомендуется для удобства фильтрации.

### Настройки скрипта

В начале файла можно изменить:

```javascript
// URL задеплоенного сервиса
const API_URL = "https://backlink-checker.up.railway.app/api/check-single";

// Домены, которые ищем на страницах гест-постов
const TARGET_DOMAINS = ["101rtp.com", "101rtp.nz", "101-rtp.nz"];
```

Если нужно проверять другие домены — добавь их в массив `TARGET_DOMAINS`.

Каждая проверка индексации — **$0.01** (DataForSEO). Проверка наличия ссылки — бесплатно.
